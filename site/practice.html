HTML Block and Inline Elements

Block-level Elements

A block-level element always starts on a new line, and the browsers automatically add some space (a margin) before and after the element.
Two commonly used block elements are: <p> and <div>

Types of Important tags

  1. <div>

    • The <div> tag defines a division or a section in an HTML document.
    • The <div> tag is used as a container for HTML elements - which is then styled with CSS or manipulated with JavaScript.
    • It is a non-semantic element, meaning it does not describe the content it contains to browsers or assistive technologies.
    • The <div> tag is easily styled by using the class or id attribute.
    • Any sort of content can be put inside the <div> tag!

    Note: By default, browsers always place a line break before and after the <div> element.

  2. <span>

    • The <span> element is an inline container used to mark up a part of a text, or a part of a document.
    • The <span> element has no required attributes, but style, class and id are common.
    • When used together with CSS, the <span> element can be used to style parts of the text.

    Example: <p>My mother has <span style="color:blue;font-weight:bold;">blue</span> eyes and my father has <span style="color:darkolivegreen;font-weight:bold;">dark green</span> eyes.</p>
    My mother has blue eyes and my father has dark green eyes.

  3. <section>

    • The <section> e tag defines a section in a document.
    • This is a semantic element that represents a standalone section of a web page with a distinct topic or purpose.
    • It provides semantic meaning to the content, indicating that the elements within it are related and form a cohesive unit. This benefits search engines and screen readers, improving accessibility and SEO.
    • A general rule of thumb is that a <section> is appropriate if its content would be listed explicitly in the document's outline (e.g., in a table of contents).

    Examples of where a <section> element can be used:

    • Chapters
    • Introduction
    • News items
    • Contact information

Class & Name Attribute

class Attribute

  • Purpose: The class attribute is used to group multiple HTML elements that share common styling or behavior.
  • Uniqueness: A class name can be applied to multiple elements on a single page, and a single element can have multiple classes (separated by spaces).

name Attribute

  • Purpose: The name attribute is primarily used with form elements (like <input>, <select>, <textarea>) to identify the data submitted to a server.
  • Uniqueness: Unlike id, the name attribute is not required to be unique across the entire page, especially within a form where multiple elements might share the same name (e.g., radio buttons).

In Summary:

class: is for styling and grouping elements based on shared characteristics. More Info
name: is for identifying form data when submitted to a server and for grouping related form controls. More Info

Flow & Phrasing Content

Flow content

  • Purpose: To create the overall layout and structure of a document.
  • What it is: The broader category of HTML content that forms the main structure of a page.
  • Examples:
    • Paragraphs (<p>)
    • Headings (<h1> to <h6>)
    • Sections (<section>)
    • Articles (<article>)
    • Tables (<table>)
    • Generic containers (<div>)

Phrasing content

  • Purpose:To define the meaning and formatting of the text itself.
  • What it is: A subset of flow content that focuses on text and its inline semantics.
  • Examples:
    • Text and inline formatting tags like <b> and <i>
    • Images (<img>)
    • Links (<a>)
    • Articles (<article>)
    • <span>
    • <em>
